Public showcase — ranked design options

The recruiter-facing screen, designed four ways and ranked. This is the deliverable that gets design sign-off and becomes the artifact for the U6 recruiter-validation gate. The bar is dual-standard: Apple-Design-Award craft and enterprise-credible. The reader is a recruiter or hiring manager — often non-technical, roughly a three-second first read. Numbers throughout are realistic and modest on purpose (hundreds of downloads, single-digit daily deltas, small star counts); the whole point is that modest-but-real reads as credible and intentional, never sparse.

The three pre-build statements

Per the frontend-design protocol: written before any screen, so the direction is committed and redirectable up front. These hold across all four ranked options — the options differ in composition, not in voice.

1 · Visual thesis

Quiet editorial confidence — warm bone paper, an optical serif (Fraunces) carrying gravitas, tabular monospace numerals that read like instruments, one disciplined "swell" teal for momentum.

The feeling is a well-kept lab notebook, not a SaaS dashboard: the craft of the page is itself the proof that this person ships real, careful work. Restraint is the flex.

2 · Content plan

Above the fold: identity line → one large absolute aggregate anchor (total downloads + total stars across all projects) → momentum as a supporting modifier → a single backfilled growth curve that is real at cold-start.

Below: the project ledger (a handful of real projects, ranked), then the killer-project deep cut (citegeist). Footer: how the numbers are sourced + "tracking started" honesty note.

3 · Interaction plan

Hero curve draws once on load (path-length reveal, ~900ms, then still). Aggregate number counts up from a near value (~600ms) so the eye lands on it. Sparkline rows lift 1px with a shadow on hover/focus and reveal a portal tooltip.

All three are gated by prefers-reduced-motion. Nothing loops; nothing decorates. Motion explains where the eye should go, then gets out of the way.

One signature accent (swell teal — growth, the ocean-swell metaphor behind the name). Amber is the only second hue and it is never decorative and never a failure color: it exclusively marks a young, not-yet-backfillable signal ("tracking started"). Neutrals are warm-tinted ink — no pure #000/#fff. Body text ink-900 on paper clears WCAG AA (≈13.5:1); swell-700 on paper ≈ 6.1:1.

When this wins

This is the default recommendation. It wins when the recruiter is non-technical and time-poor: the eye lands on a real four-figure absolute (2,418 downloads · 247 stars) in well under three seconds, momentum sits right beside it as support — never a bare percentage — and the single backfilled curve proves the growth is real on day one, before any perishable signal has accrued. The ranked ledger then answers the "is this breadth or one lucky repo?" question a single hero can't. Editorial serif + instrument numerals read as craft, which is itself a hiring signal.

It is also the most honest cold-start design: the amber boundary and "tracking since Apr 2026" treatment turn the product's biggest weakness (young download data) into a visible signal of rigor, satisfying AE1/AE3/AE6 directly.

Rank 4 · Rejected The Percentage Marquee

Rendered deliberately as the anti-pattern the brainstorm and KTD12 warn against, so the failure is concrete. It leads with a huge bare growth percentage, foregrounds perishable velocity, and has nothing honest to show at cold-start. Kept in the set as a guardrail, not a candidate.

Why this is rejected (never ships)

"+100%" is two downloads becoming four. A percentage with no denominator reads as evasive to a recruiter — exactly the failure KTD12 and AE3 call out. The honest absolute (2,418 · 247) is both available immediately and far more credible; hiding it behind a hype number actively damages trust.

Perishable-velocity-first breaks at cold-start. Views/clones live in a 14-day owner window and don't exist on day one, so the strongest tile is a flat "no data yet" — the embarrassing empty card the whole product exists to avoid (AE6). It also violates the design bar on empty states (a flat line is not a considered empty state) and on hierarchy (one giant decorative number, emoji as content). Documented here so no future session reaches for the "big % hero" by reflex.

Resolved decisions the doc review flagged these as undecided

Each item the plan called out as open for U5 is decided here, with the reasoning, so U6 tests a fully-specified design and U9/U10 build against a fixed target.

First-screen information architecture + navigation

Single long scroll, anchor nav — Overview → Projects → killer project (citegeist) → sourcing footer. No app shell, no sidebar: this is a document a recruiter reads top-to-bottom, not a tool they operate. The top bar is sticky and thin; its links are in-page anchors. The killer project gets a named anchor (/citegeist deep route) so it is independently shareable (R15). Order is a deliberate funnel: absolute proof → breadth → depth → method.

Per-project card content hierarchy

Fixed read order, left→right: identity (badge + name + one-line deck) → trend (12-week star sparkline) → stars → downloads. The lead signal is the absolute count with a small supporting delta beneath it (never a bare %). Stars lead over downloads because stars are backfilled and therefore always present; a young project shows the "tracking · N wk" chip in the downloads slot instead of a number — the slot is never blank (mirrors the sponsor-row "never absent" discipline). Whole row is one tap target; no nested links.

Tooltip interaction — hover + keyboard focus + mobile tap
Mar 2026 · stars128

One portal-rendered tooltip (escapes any overflow:hidden chart clip — the dropdown-portal-stacking learning), dark surface, mono numeral + uppercase label. Desktop: follows the nearest data point on hover with a focus crosshair. Keyboard: the chart is one focusable element; arrow keys step through points and the tooltip tracks focus (visible focus ring). Mobile: tap snaps the tooltip to the nearest point; tap-elsewhere dismisses — no hover-only affordance. An offscreen <table> mirrors the series for screen readers.

Dark mode — recommend LOCK TO LIGHT for v1

Ship light-only for v1. Three reasons: (1) the cdo bar treats dark mode as its own design, not an inversion — the data-viz palette (categorical luminance balance, area-fill opacities, the amber tracking-boundary against paper) would need a full second tuning pass to stay credible, and that work doesn't move the recruiter metric; (2) the warm-bone editorial paper is the brand's point of view — a dark variant dilutes the "lab notebook" thesis on first impression; (3) it is cheap to add later precisely because the token architecture isolates it — a future dark theme redefines only the Layer-2 semantic aliases and a dark data-viz set, with Layer-1 scale and all primitives untouched. The cost of waiting is one media query later; the cost of doing it now is a half-built second palette that undercuts the launch. Honest naming of the cost, per Josh's standing principle: this defers real work, but the deferral is clean and reversible, not debt that compounds.
